### 🔴 HIGH Issues (2)

#### 1. Potential Race Condition in `saveAllPermissionStates()`
**Location:** `MainViewModel.kt:1454`

The method uses a sequential `for` loop calling `savePermissionState()` for each permission. While `ioDispatcher` is limited to 1 thread, there's no strict guarantee of sequential execution when multiple callers invoke this method simultaneously.

**Recommended Fix:**
```kotlin
private suspend fun saveAllPermissionStates(missing: List<RequiredPermission>) {
    withContext(ioDispatcher) {  // Wrap entire loop
        for (permission in RequiredPermission.entries) {
            val granted = !missing.contains(permission)
            PermissionManager.savePermissionState(permission, granted)
        }
    }
}
```

#### 2. `runBlocking` Usage in Instrumented Tests
**Location:** `PermissionManagerInstrumentedTest.kt:152, 185, 214`

Tests use `runBlocking` to call `savePermissionState()` which is now suspend. This blocks the test thread and may mask concurrency issues.

**Recommended Fix:**

    }

}
